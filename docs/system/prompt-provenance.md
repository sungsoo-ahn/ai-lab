# Prompt Provenance

Prompts are part of the scientific record. A future maintainer should be able to inspect what an LLM run was asked to do, which work units it touched, and how that prompt relates to the resulting reports.

## Default Policy

Exact run prompts are local run artifacts by default. Public documentation may describe the prompt's purpose and local path, but should not copy raw run prompts into GitHub Pages unless explicitly approved after a privacy review.

This keeps the public manual readable while preserving enough local evidence to audit agent behavior.

## Storage Convention

Store prompts under the run directory:

```text
tasks/active/<task_id>/scientists/<scientist_id>/runs/<run_id>/
├── prompt-manifest.yaml
└── prompts/
    └── <prompt_id>.md
```

The prompt manifest records:

- `prompt_id`
- local prompt file path
- source command or launcher
- sensitivity
- related work units

## Recording A Prompt

Record a prompt before or immediately after launching an LLM run:

```sh
bin/ai-lab prompt record <task_id> <scientist_id> <run_id> <prompt_id> \
  --source <launcher-or-command> \
  --sensitivity local \
  --work-unit <work_unit_id> < prompt.md
```

For a prompt that already exists in a file:

```sh
bin/ai-lab prompt record <task_id> <scientist_id> <run_id> <prompt_id> \
  --from-file prompt.md \
  --source <launcher-or-command>
```

## What Public Docs Should Say

Public scientist and work-unit manuals should summarize:

- which prompt artifact controls the run;
- where the local prompt manifest lives;
- which work units the prompt covered;
- whether the prompt was generated by a helper, copied from a connector, or written manually.

They should not paste long raw prompts into public pages. Use the local artifact path instead.

## Current Example

The BTC overnight run stores its historical orchestration prompt at:

```text
tasks/active/btc/scientists/btc_autoresearch_v1/runs/overnight-2026-06-09/prompts/orchestration.md
```

The manifest is:

```text
tasks/active/btc/scientists/btc_autoresearch_v1/runs/overnight-2026-06-09/prompt-manifest.yaml
```

That prompt was captured from the run log after the workspace migrated from `agent-system` to `ai-lab`; the path names inside the prompt are historical context, not the current canonical layout.
